Git的使用

linux: 
1. yum install git 
2. git  clone  .... 
3. git add  file  
4. git commit file  
5. git push 
6.若有冲突：git pull

####Git冲突是如何产生的
若两个人在更新下来的工程文件中，建立各自的文件并且各自干各自的事情，则一定没冲突。
若两人在更新下来的文件里,在同一文件的不同地方各自个各自的事情也没有冲突。
若两人在更新下来的文件里,在同一文件的相同地方修改则发生冲突。
先提交的那个人没事，后面提交的那个会报错，提示一个版本已经提交了相同的引用。
git pull 后两人的代码合并，两人沟通，看留谁的代码，手动删除不用。

####Git回到过去回到未来
过去：1. git log 2. git reset –hard 40位数字
未来：1. git reflog 2. git reset –hard 仅仅几位数

####建立里程碑
github:release

####建立分支
新的功能已经开始开发，但是以前的出现了bug ,为了不影响新的功能的开发则在之前的里程碑开一个分支。
创建分支：git branch bug # bug 为分支的名字
查看分支：git branch
切换分支：git checkout bug
合并：软件合并；github网站合并；多用软件少用命令行，效率源于工具的使用。
